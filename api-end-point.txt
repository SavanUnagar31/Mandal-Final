Mandal Backend API Endpoints

# Authentication Endpoints
- POST /api/v1/auth/register
  Description: Register a new user with name, email, mobile, and password. Sends verification code.
  Authentication: None
  Response: { success: true, data: { userId: number }, message: "Registration successful, verification code sent" }
  Status: 201 (Created), 400 (Bad Request)

- POST /api/v1/auth/login
  Description: Login a user with email and password. Returns JWT token.
  Authentication: None
  Response: { success: true, data: { token: string }, message: "Login successful" }
  Status: 200 (OK), 401 (Unauthorized), 403 (Forbidden)

- POST /api/v1/auth/verify
  Description: Verify user's email with verification code.
  Authentication: None
  Response: { success: true, data: user, message: "Email verified" }
  Status: 200 (OK), 400 (Bad Request), 404 (Not Found)

- POST /api/v2/auth/register
  Description: [v2] Same as v1, placeholder for future updates.
  Authentication: None
  Response: { success: true, data: { userId: number }, message: "Registration successful, verification code sent" }
  Status: 201 (Created), 400 (Bad Request)

- POST /api/v2/auth/login
  Description: [v2] Same as v1, placeholder for future updates.
  Authentication: None
  Response: { success: true, data: { token: string }, message: "Login successful" }
  Status: 200 (OK), 401 (Unauthorized), 403 (Forbidden)

- POST /api/v2/auth/verify
  Description: [v2] Same as v1, placeholder for future updates.
  Authentication: None
  Response: { success: true, data: user, message: "Email verified" }
  Status: 200 (OK), 400 (Bad Request), 404 (Not Found)

# Mandal Endpoints
- POST /api/v1/mandals
  Description: Create a new mandal with name, contribution mode, amount, and interest rate.
  Authentication: JWT required
  Role: Any authenticated user
  Response: { success: true, data: mandal, message: "Mandal created" }
  Status: 201 (Created), 400 (Bad Request), 401 (Unauthorized)

- GET /api/v1/mandals/:id
  Description: Retrieve a mandal by ID.
  Authentication: JWT required
  Role: Member or Admin of the mandal
  Response: { success: true, data: mandal, message: "Mandal retrieved" }
  Status: 200 (OK), 401 (Unauthorized), 403 (Forbidden), 404 (Not Found)

- PUT /api/v1/mandals/:id
  Description: Update a mandal's details.
  Authentication: JWT required
  Role: Admin of the mandal
  Response: { success: true, data: mandal, message: "Mandal updated" }
  Status: 200 (OK), 400 (Bad Request), 401 (Unauthorized), 403 (Forbidden), 404 (Not Found)

- DELETE /api/v1/mandals/:id
  Description: Delete a mandal.
  Authentication: JWT required
  Role: Admin of the mandal
  Response: { success: true, data: null, message: "Mandal deleted" }
  Status: 200 (OK), 401 (Unauthorized), 403 (Forbidden), 404 (Not Found)

- POST /api/v2/mandals

- GET /api/v2/mandals/:id
  Description: [v2] Same as v1, placeholder for future updates.

  Role: Member or Admin of the mandal
  Response: { success: true, data: mandal, message: "Mandal retrieved" }
  Status: 200 (OK), 401 (Unauthorized), 403 (Forbidden), 404 (Not Found)

- PUT /api/v2/mandals/:id
  Description: [v2] Same as v1, placeholder for future updates.

  Role: Admin of the mandal
  Response: { success: true, data: mandal, message: "Mandal updated" }
  Status: 200 (OK), 400 (Bad Request), 401 (Unauthorized), 403 (Forbidden), 404 (Not Found)
